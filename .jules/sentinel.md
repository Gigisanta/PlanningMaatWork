## 2024-05-24 - [Critical XSS Risk in User-Editable Iframe Previews]
**Vulnerability:** The application renders user-editable HTML content directly into an `iframe` via the `srcDoc` attribute without any restrictions (no `sandbox` attribute). This allowed stored/reflected Cross-Site Scripting (XSS), meaning any malicious scripts embedded in the `editableHTML` could execute in the context of the main application.
**Learning:** This vulnerability existed because the architecture requires the main application (via `html2pdf.js` / `html2canvas`) to reach into the `iframe`'s DOM to generate a PDF export. A completely sandboxed `iframe` without `allow-same-origin` prevents this cross-document access, leading developers to sometimes omit the `sandbox` attribute entirely when they encounter PDF generation errors, unknowingly introducing a critical XSS vector.
**Prevention:** To avoid this, `iframe` elements displaying user-controlled or generated HTML must always use the `sandbox` attribute. For this specific application's PDF generation architecture to function correctly while maintaining security, the `sandbox` configuration must explicitly be `sandbox="allow-same-origin allow-popups"`. This configuration explicitly omits `allow-scripts`, thereby preventing the execution of any malicious JavaScript embedded in the user's HTML, while `allow-same-origin` satisfies the requirement for the parent window's PDF generation library to access the child frame's DOM.
