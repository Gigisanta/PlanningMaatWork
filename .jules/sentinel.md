## 2024-05-16 - [XSS via Iframe srcDoc]
**Vulnerability:** The application renders user-editable HTML content using an `iframe` with the `srcDoc` attribute but lacks the `sandbox` attribute. This allows arbitrary script execution within the iframe context if malicious scripts are injected into the HTML.
**Learning:** `srcDoc` iframes default to full permissions if `sandbox` is omitted, contrary to what some might expect regarding built-in isolation. Since the parent document extracts HTML from this iframe for PDF generation (`iframe.contentDocument`), full sandboxing (`sandbox=""`) breaks the application because it prevents same-origin access.
**Prevention:** Always apply the `sandbox` attribute to iframes rendering untrusted or editable content. For this specific pattern (editable content that needs DOM extraction by parent), use `sandbox="allow-same-origin allow-popups"` which prevents script execution (`allow-scripts` is omitted) while allowing the parent to read the DOM for PDF generation and allowing legitimate outgoing links to open.
