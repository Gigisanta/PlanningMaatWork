## 2025-03-08 - Prevent XSS in Editable HTML Previews
**Vulnerability:** The `PortfolioPreview` component rendered user-editable HTML directly into an `iframe` via `srcDoc` without any `sandbox` attribute. This allowed full execution of arbitrary JavaScript if a user entered `<script>` tags or inline event handlers into the editor, creating a Cross-Site Scripting (XSS) vulnerability.
**Learning:** React's built-in XSS protections (like escaping in `{}`) do not apply when rendering raw HTML into an `iframe` using `srcDoc` or when using `dangerouslySetInnerHTML`. Since the application allows users to edit the raw HTML of their portfolio before downloading/previewing, the iframe must be explicitly sandboxed to prevent script execution.
**Prevention:** Always use the `sandbox` attribute on `iframe` elements that render untrusted or user-editable content. In this case, `sandbox="allow-same-origin allow-popups"` was added. This intentionally omits `allow-scripts` to neutralize the XSS threat, while retaining `allow-same-origin` (needed for PDF generation/DOM access) and `allow-popups` (for functional links).
